Что должно получиться в итоге?
    Строковое представление объекта с примерами и локализованными строками документации.
    Представлением должен заниматься такой объект, на вход которому подаются объекты для представления, а на выходе получаем представление этого объекта. Причём, если составителю документации не нравится поведение базового объекта, то объект представления должен поддерживать внесение изменений.
    Способы реализации:
    1) Класс с методами для представления сущностей(Repr).
        + 
            возможность менять поведение наследованием
        -
            сложности при добавлении новых обработчиков объектов
            дополнительная логика для передачи объектов в методы

    2) Диспетчер с разделяемыми данными(Disp)
        +
            Единая точка входа для всех объектов
            Разделяемые данные дают возможность управлять представлением 

        -
            Нет возможности управлять наследованием

        +|-
            Объявления обработчиков децентрализованы

    3) Гибридный 
        Методы и разделённые данные находятся в классе Repr. 
        Точка входа и логика выбора метода для представления в диспетчере Disp.

    Вопрос: все объекты включаю в себя другие объекты, которые также нуждаются в представлении. Реализовать рекурсивный обход или предоставлять для представления объекты, которые включаю в себя только строки - уже прошедшие этап представления объекты?
    Ответ: Repr инкапсулирует логику представление объекта, который состоит из строк. Disp - логика направления объекта нужному методу Repr. И над Disp находится функция(obj2string), которая получает объект и, если он включает вложенные объекты, то отправляет его на представление отдельно (рекурсивный обход).
    Объект, который получает функция obj2string, уже должен включать локализованную версию документации, дополнительные пояснения к параметрам функций. Она должна быть полностью подготовлена к представлению.
    Неподготовленный объекты выгружаются на основе кода, подготовленные объекты поступают в объект представления. Разницу между ними должен подготовить человек вручную, или она должна готовиться на основе правил. Разница (или правила) должна составляться переносимым образом. (Формат json?)
